GETTING STARTED

The downloaded package should contains four directories:

1. Fireworks PNG: here are the original layered png files
2. HELP: the documentation file that you actually reading now:)
3. html: this folder contains the entire tamplate with all of its elements
4. PSD: here you can find the psd files, but they are generated with fireworks ( Because I working with FW )

Creating a backup folder for the original template files is highly recommended before start the customizing. If something goes wrong, those backup files can be very useful.

To start the work with your new website, just place the html folder in any directory on your computer, open it, and open the index.htm file. If you doubleclick on the file, it will be opened normally in your default browser, but you need to open it with one of the popular text editors too. I prefer Context, but the Notepad++ is can be an another option.

Which is the best browser for the developement?
I using Chrome, because it comes with the best developer tools that I've ever seen ( at least this is my opinion ). But.. before start the work, you need to know that the ajax functions doesn't work in chrome ( in one word: the portfolio and service items won't be loaded )... except you use a LAMP emulator on your local computer ( for example: xampp (for windows), appserw (for windows) and mamp (for mac) ). In one word: if you have a webserver, I recommend Chrome, otherwise the Firefox will be a best solution for you.

FILES IN THE PACKAGE

This theme comes with three html files, but you can find some other file types such as javascript, and php. All of these are listed and shortly explained below:


HTML:

index.htm - The main tamplate file in the defaul style.
portfolio_item.htm - A sample portfolio item that contains a slider for the project images. This file is loaded by ajax.
sample_service.htm - A sample service item. Contains text only. This file is loaded by ajax just like the portfolio contents.


CSS:

styles/styles.css - The default stylesheet, and a very important one, because it contains all of the general styles for the layout, and typography, so please create a backup file, before start to modify it!

styles/settings.css - This file is built to make the customization easier. It contains the colors and font families only, but it is deactivated in defeault case.

styles/ie8.css - This file makes the theme ie8 compatible.


JAVASCRIPT:

jquery.js - The JQuery core, very very important, and should be the first.
effects.js - An effect library for JQuery.
easing.js - An other effect library for JQuery.
flexslider-min.js - The script behind the responsive slider.
mobilemenu.js - This file responsible for the navigation on mobile devices.
tipTip.js - Tooltip plugin.
hoverIntent.js - Extends the jquery library with the hoverIntent() event.
quicksand.js - This plugin works when you click on a link in a portfolio filter.
custom.js - This script contains the specific codes for this template.


PHP:

phpmailer.php - A robust PHP email transport class.
mail.php - Here you can add your email address to the contact form.


IMAGES:

You can find some great icons in the images/service_icons folder. These can be useful when you want to show your services on the site. The theme contains a few cool social icons too in the images/social_icons folder. Unfortunately I can't include the images from the demo, but you may download some of them for free. GENERAL STRUCTURE

This theme has a responsive layout, with one main column. All of the elements are nested within a div with an id of "wrapper". There are six main sections within the "wrapper":

1) header
2) slider_wrapper
3) portfolio
4) services
5) about
6) features
7) contact

The "features" div is created for demostrates the grid system of the template on the demo page, so I recommend you to remove it. Of course, if you don't need one of the other sections ( for example the about ), you may remove it.

NAVIGATION

There are two navigations in the template. The main menu can be found under the logo, and always visible. The hidden menu appears only when the main menu has disappeared (try to scroll down the page). Both of them are based on a simple unordered list, with left floated items. It is important to know that the page scroll effects gets their target from the href and value attributes. So, you may edit the text label of the menu items, but please don't edit the href attributes! 

                <div id="navigation">

                     <!-- Menu items -->

PORTFOLIO

I think the portfolio system is the most interesting part of the theme. Users can filter the elements ( thanks to the Quicksand plugin ) and the single item page loaded via ajax.

Below you can see the html structure of the portfolio. In <ul id="portfolioFilter">, each of the anchor tags has a class. This class tells the script what you are filtering when clicking on. All of the portfolio elements nested as a list item within the
<ul class="sortablePortfolio">. Each of these has a 'data-type' attribute. The value of this attribute must be consistent with one of the anchors in <ul id="portfolioFilter">. For examaple, if you have a filter anchor with a class="webdesing", and you want to add an element to the webdesign category, the data-type must be the same: webdesign.

All of the portfolio elements has a data-id attribute too. That tells the script how to ordering elements. Do not create two elements with the same data-id.

There are an empty div with a class of "item_container". It can be found under the filter. This div will be the container for the item details. When the user clicks the "more information" button the info will be loaded into this div, so please don't remove it and leave it empty.

The size of the thumbnails is 520x520 in order to keep their look sharp on retina devices. Please don't worry about the bandwidth. Bigger resolution doesn't have to mean bigger file size ( useful info here )! An optimized jpeg with 520x520 resolution is lighter than an avarage 260x260 image without optimization. If you don't know how to compress a jpeg file, feel free to visit this link:

Jpeg Optimizer

I recommend you to create the 520x520 thumbnails, then optimize them with a 80 compression level. The "Resize Photo" function is unused in this case, so feel free to uncheck it.

2) Contact form

Important: Do not remove or modify the ids and name values of the form elements, because they are necessary in the validation process. You may modify the value attributes, but if you add a new value to the selected element, you have to modify the relevant variable in the custom.js too. 

                <div id="contact_form">
               
                     
                     <input type="text" name="name" value="YOUR NAME *" id="form-name" />

                     
                     <input type="text" name="mail" value="YOUR EMAIL ADDRESS *" id="form-mail" />
               
                     
                     <textarea name='message' rows="5" cols="62" id="form-message">YOUR MESSAGE *</textarea>
               
                     
                     <button type="submit" id="button" >Go</button>

                     
                     <div id="result"></div>

                </div>


You can customize the validator with the following options in the custom.js 
                var name_value   = 'YOUR NAME *'; //must be the same as the value of the name field
                var mail_value   = 'YOUR EMAIL ADDRESS *'; //must be the same as to the value of the email field
                var message_value= 'YOUR MESSAGE *'; //must be the same as the content of the textarea

                var missing_name = 'Please tell us your name!'; //error message, if the name field is empty
                var missing_mail = 'Please let us know your email address!'; //error message, if the mail field is empty
                var invalid_mail = 'Oh no! Invalid email address!'; //error message, if the user's email address is invalid
                var missing_message = 'Please write us something beautiful!'; //error message, if the textarea is empty

                var error_color   = '#990000'; //text color of the error messages
                var default_color = '#333333'; //default text color of the contact form

To activate your contact form open the mail.php, and modify the following variables:

The theme uses three css files and all of them can be found in the styles folder. The stlye.css is the main css file, this contains all of the styles. WORKING WITH THE FONTS

Near the system fonts the theme contains four families from the google font library: Brawler, Vollkorn, Quicksand and Droid+Sans. You can add these types to any element in the document ( if you open the setting.css, you will see how it works ). If you don't like the available fonts, feel free to add more families to your site. The styleswithcer on the demo page can help you to make the decision. If you don't know, how to use the Google Font Library, visit the official help page.
